Installing Neon

Prerequisites

Before installing Neon, you should unzip the latest Jini distribution from the Apache River project, as of writing this is Apache River 2.1.1

Running the installer

Download the latest Neon binary from here, you can then double-click the downloaded jar, to run the installer. Follow the prompts as normal, until you are asked to Configure the installation.

Configuring

When you run the post install, you should choose Custom configuration, select the location of where you unzipped the Jini distribution above, as the Jini installation directory, and select the jar [jini installation root]/lib/serviceui.jar for the serviceui location.i.e. C:\Development\Java\apache-river-2.1.1for Jini location and C:\Development\Java\apache-river-2.1.1\lib\serviceui.jar (the installer will append 'lib' to the end of the Jini installation directory). You should only have to do this once, the post-installer stores the locations you've chosen.

After pressing Next you'll be asked to download other prerequisites for Neon, these are automatically downloaded for you, unless you tell the Installer that you've already got them on your system.

If you need to download any of the files, click Locate, and wait until the download progress dialog has completed.

You will see a page asking you for any additional libraries that you want to use, this will form the classpath for Neon's scripts. Jars will be added here as the prerequisites are downloaded and unarchived.

Base configuration

Neon ships with a base setup for Jini, that matches the default configuration given - if you wish to run Neon as a separate Jini group, just accept the defaults, and move on. Otherwise you can change the defaults to match your environment.

Codebase

If you're using the supplied Jini setup, click on the '...' button next to 'Codebase URL' enter the 'http://localhost:8085' into the popup dialog, and then click yes when asked if you want to copy the codebase files to a directory.

In the file chooser, select the directory where you installed the jini distribution and choose the lib-dl directory.

Now click the 'Generate' button.And it's all done.

Testing it all out

Open a couple of Shells or command prompts, and go to where you installed Neon, and go into the jini setup directory, run the start-ws file and the start-phoenix file for your system (.bat for windows, .sh for Linux, Unix and MacOSX)

Should everything go okay you should see activity in both windows and no exceptions being thrown. Now run start-base-setup, again everything should go okay, and some info should be seen in the phoenix window.

At this point, all the Jini infrastructure that Neon requires is now running, and you can start the runneon file in the neon installation root directory, and hopefully, after a lot of output going past, you'll see 'Neon initialised'.
